## Fixes Applied This Session (2026-07-15)

### 1. `ros2_ws/toolchain.cmake` — Python cross-compile variables (REALLY added now)
After `PKG_CONFIG_EXECUTABLE` block:
- `PYTHON_LIBRARY/PYTHON_LIBRARIES/Python3_LIBRARY/Python3_LIBRARIES` →
  `${DOGBOT_SYSROOT}/usr/lib/python3.10/config-3.10-${DOGBOT_TARGET_TRIPLET}/libpython3.10.so`
  (config-dir path on purpose: `usr/lib/<triplet>` is an implicit linker dir and CMake
  strips the sysroot prefix there, generating deps on nonexistent host paths)

- `PYTHON_SOABI` → `cpython-310-${DOGBOT_TARGET_TRIPLET}` (otherwise FindPythonExtra
  computes SOABI from HOST python → x86_64-tagged .so that target python won't import)
- All `CACHE ... FORCE`, plus EXISTS sanity check (FATAL_ERROR if sysroot python missing)

## Docker 构建代理问题（仅重建镜像时相关）

### 问题
Docker daemon 配置了 `HTTP_PROXY=http://127.0.0.1:7897/`
（`/etc/systemd/system/docker.service.d/proxy.conf`）——这只对 daemon 拉镜像有效。
构建 `RUN` 步骤里容器访问不到代理：`127.0.0.1` 是容器自身回环；`172.17.0.1:7897`
不通，因为 clash-verge `allow-lan: false`（mixed-port 只绑 127.0.0.1）。

### 前置条件（若走代理方案）
clash-verge 打开「局域网连接 / allow-lan」→ 7897 监听 0.0.0.0，容器才能经
`172.17.0.1:7897` 访问。验证：
`docker run --rm alpine sh -c 'nc -z -w3 172.17.0.1 7897 && echo ok'`

### CURL_PROXY 方案（Dockerfile 改动至今【未落地】）
不能全局 `--build-arg HTTP_PROXY=...`（apt 会走代理报 Connection refused）。
用专用 `ARG CURL_PROXY=`，只给 curl/wget/rosdep 显式加代理：
- `dogbot-base`/`dogbot-develop`/`dogbot-runtime` 三段头各加 `ARG CURL_PROXY=`
- unison/nodesource/neovim curl: `curl ${CURL_PROXY:+-x ${CURL_PROXY}}`
- LLVM key/cmake/ohmyzsh wget: `wget ${CURL_PROXY:+-e https_proxy=${CURL_PROXY}}`
- rosdep: `HTTP_PROXY= HTTPS_PROXY=${CURL_PROXY} rosdep update`
构建命令追加 `--build-arg CURL_PROXY=http://172.17.0.1:7897/`
